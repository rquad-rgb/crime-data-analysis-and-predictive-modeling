Overview
This project analyzes crime data from Los-Angeles to:
• Examine trends in crime over time
• Identify which victim groups are most affected
• Build a predictive model for arrest outcomes
The analysis follows a full data science workﬂow including data cleaning, exploratory analysis,
statistical modeling and model evaluation.
• Note: The dataset contains missing values and categorical inconsistencies that require
preprocessing.

Model Limitations
The poor performance is likely due to:
• The data is imbalanced, with far fewer arrest cases than non-arrest cases
• The variables used (age, sex, descent, and area) may not be strong predictors of whether
an arrest happens
• Some categories have very few observations, which makes it harder for the model to
learn reliable patterns.
Because of this, the model has difﬁculty correctly identifying arrest cases.
Conclusion
This project demonstrates a full data analysis process using real-world crime data.
While the exploratory analysis showed some interesting patterns, the predictive model did not
perform well. In particular, it struggled to accurately identify arrest cases, and its performance
was sensitive to the classiﬁcation threshold.
This highlights how accuracy alone can be misleading, especially when the data is imbalanced.
In the future, the model could be improved by:
• adding more relevant features
• trying different ways to handle the imbalance
• testing other types of models
Overall, this project shows the importance of carefully evaluating model performance and
understanding its limitations.
